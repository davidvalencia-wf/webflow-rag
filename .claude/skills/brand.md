# Brand Guidelines Skill

You are helping maintain brand consistency across the Webflow RAG project. Always reference and apply the brand guidelines documented in `docs/brand-guidelines.md`.

## Your Role

When working on any user-facing code, design, or content:

1. **Check brand compliance** - Ensure colors, typography, and spacing match guidelines
2. **Apply brand voice** - Write copy that matches the brand tone
3. **Maintain consistency** - Use established patterns and components
4. **Flag violations** - Alert when existing code doesn't match guidelines

## Key Guidelines Reference

> **Source of Truth**: `/Users/ryan.hodge/Projects/webflow-rag/docs/brand-guidelines.md`
>
> Always read the latest version from the docs folder for current specifications.

### When to Apply Brand Guidelines

Apply brand guidelines when:
- Creating or modifying UI components
- Writing user-facing text or error messages
- Setting up styling systems (CSS, Tailwind, etc.)
- Designing layouts and spacing
- Choosing colors or typography
- Creating animations or interactions

### Brand Compliance Checklist

Before completing any UI task, verify:
- [ ] Colors match the defined palette
- [ ] Typography uses approved fonts and scale
- [ ] Spacing follows the base unit system
- [ ] Components match established patterns
- [ ] Copy matches brand voice and tone
- [ ] Accessibility standards are met
- [ ] Responsive behavior is consistent

## Workflow

1. **Read current guidelines**: Always check `docs/brand-guidelines.md` first
2. **Apply specifications**: Use exact values from the guidelines
3. **Suggest improvements**: If guidelines are unclear or incomplete, suggest updates
4. **Document decisions**: When making brand-related decisions, note them

## Common Tasks

### Adding a New Component
1. Check if similar component exists in guidelines
2. Apply brand colors, typography, and spacing
3. Ensure accessibility compliance
4. Consider adding to component library

### Updating Styles
1. Verify changes align with brand guidelines
2. Update all instances for consistency
3. Test across different screen sizes
4. Document any new patterns

### Writing Copy
1. Match the brand voice described in guidelines
2. Use approved terminology
3. Follow capitalization and punctuation rules
4. Keep accessibility in mind (clear, concise language)

## Tips

- When in doubt, ask the user if something should be added to brand guidelines
- Prioritize consistency over personal preference
- Brand guidelines can evolve - suggest updates when helpful
- Accessibility is part of brand quality

---

**Remember**: The brand guidelines in `docs/brand-guidelines.md` are the authoritative source. Always reference them when working on user-facing features.
